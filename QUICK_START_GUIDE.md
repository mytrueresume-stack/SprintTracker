# ?? Quick Start Guide - Sprint Metrics System

## ? Get Started in 5 Minutes

### For Managers (3 steps)

#### Step 1: Create Sprint (30 seconds)
1. Navigate to your project
2. Click "Sprints" tab
3. Click "Create Sprint" button
4. Fill in:
   - Sprint Name: "Sprint 1"
   - Sprint Goal: "Your sprint objective" (optional)
   - Start Date: Select date
   - End Date: Select date (typically 2 weeks)
5. Click "Create Sprint"

#### Step 2: Start Sprint (10 seconds)
1. Find your sprint in the list
2. Click three-dot menu (?)
3. Click "Start"
4. Done! Sprint is now Active ?

#### Step 3: View Reports (Anytime)
1. Find your active sprint
2. Click "View Report" button
3. See real-time metrics, charts, and team submissions

---

### For Developers (2 steps)

#### Step 1: Find Active Sprint (5 seconds)
1. Go to Dashboard
2. Look for "Active Sprints" section
3. See your active sprint card with "Submit Work" button

#### Step 2: Submit Your Work (3-5 minutes)
1. Click "Submit Work" button
2. Fill out the form:
   - **Work Summary**: Story points and hours
   - **User Stories**: What you worked on
   - **Features**: What you delivered
   - **Impediments**: Any blockers (if any)
   - **Achievements**: What you accomplished
   - **Learnings**: What you learned
   - **Next Goals**: Plans for next sprint
   - **Recognitions**: Thank your teammates
3. Click "Save Draft" (to continue later) OR "Submit Work" (final)

## ?? Form Filling Guide (Developers)

### Work Summary (Required)
```
Story Points Planned: [15]      # What you planned to complete
Story Points Completed: [12]   # What you actually completed
Hours Worked: [40]   # Total hours spent
```

### User Stories (Add as many as you worked on)
```
Click [+ Add Story]

Story ID: PROJ-123               # Your tracking ID
Title: User Login Feature       # What you worked on
Story Points: 5     # Points for this story
Status: Completed             # Completed/In Progress/Testing
```

### Features (Add what you delivered)
```
Click [+ Add Feature]

Feature Name: JWT Authentication
Description: Implemented token-based authentication
Module: Security              # Which part of system
Status: Delivered         # Delivered/Partially Delivered
```

### Impediments (Only if you had blockers)
```
Click [+ Add Impediment]

Description: API documentation was unclear
Category: Technical # Technical/Process/External
Impact: Medium                  # Low/Medium/High/Critical
Status: Resolved                # Open/Resolved/Escalated
Resolution: Discussed with team lead
```

### Achievements (Free text)
```
What did you accomplish?

Example:
- Implemented JWT authentication
- Created user profile page
- Fixed 3 critical security bugs
- Mentored junior developer
```

### Learnings (Free text)
```
What did you learn?

Example:
- Learned React hooks in depth
- Better understanding of JWT tokens
- Improved at estimating story points
- Database optimization techniques
```

### Next Sprint Goals (Free text)
```
What do you plan to work on?

Example:
- Complete payment integration
- Start working on admin dashboard
- Help with code reviews
- Improve unit test coverage
```

### Team Recognitions (Add teammates to thank)
```
Click [+ Add Recognition]

Appreciated: Jane Smith
Reason: Helped debug a critical production issue
Category: Teamwork          # Teamwork/Technical/Leadership
```

### Additional Notes (Free text)
```
Any other comments?

Example:
Great sprint overall. Team collaboration was excellent.
Would benefit from better documentation.
```

## ?? Pro Tips

### For Managers:

**? DO:**
- Create sprints with clear, achievable goals
- Start sprints at the beginning of the period
- Monitor reports regularly during the sprint
- Review submissions before completing sprint
- Share velocity metrics with the team
- Address impediments quickly

**? DON'T:**
- Don't wait to start the sprint
- Don't forget to complete the sprint on time
- Don't ignore impediments
- Don't delete active sprints

### For Developers:

**? DO:**
- Submit your work before the sprint ends
- Be honest about story points completed
- Document impediments (helps the team)
- Give team appreciations (builds morale)
- Save draft frequently if filling gradually
- Add detailed user stories

**? DON'T:**
- Don't wait until the last day to submit
- Don't inflate story points
- Don't leave achievements section blank
- Don't forget to thank teammates
- Don't submit without reviewing your input

## ?? Common Scenarios

### Scenario 1: Developer New to Sprint
**Q:** "I don't see any active sprint. What do I do?"
**A:** Contact your manager to start the sprint. Once started, you'll see it immediately.

### Scenario 2: Developer Wants to Edit Submission
**Q:** "I already submitted but want to change something."
**A:** You can only edit if you saved as draft. Once submitted, it's final. Contact your manager if you need to make corrections.

### Scenario 3: Manager Wants to See Who Submitted
**Q:** "How do I know who has submitted their work?"
**A:** 
1. Go to Sprint Report
2. Click "Team Performance" tab
3. See submission status for each member

### Scenario 4: Manager Wants to Edit Sprint Dates
**Q:** "Can I change the sprint end date?"
**A:** 
1. Yes! Find your sprint
2. Click menu (?) ? Edit
3. Change end date
4. Save

### Scenario 5: Developer Forgot to Add a Story
**Q:** "I submitted but forgot one story. Can I add it?"
**A:** Once submitted, form is locked. Save as draft next time to review before final submit.

## ?? Troubleshooting

### Problem: "I don't see Submit Work button"
**Solution:**
1. Check if sprint is Active (not Planning or Completed)
2. Refresh your browser
3. Verify you're in the project team
4. Contact your manager

### Problem: "My submission didn't save"
**Solution:**
1. Check your internet connection
2. Try clicking "Save Draft" instead of "Submit"
3. Check browser console for errors (F12)
4. Refresh and try again

### Problem: "Charts not loading in report"
**Solution:**
1. Wait a few seconds for charts to render
2. Refresh the page
3. Check if developers have submitted data
4. Try different browser

### Problem: "Can't edit my submission"
**Solution:**
- Once submitted, submissions are final (locked)
- Use "Save Draft" if you need to edit later
- Contact manager if urgent corrections needed

### Common Questions

**Q: When should I submit my work?**
A: Anytime during the sprint, but before it ends. Earlier is better!

**Q: Can I submit multiple times?**
A: You can save draft multiple times, but only submit once (final).

**Q: What if I don't complete all planned points?**
A: Be honest! Enter actual completed points. It helps team improve estimates.

**Q: Should I fill all sections?**
A: Work Summary is required. Others are optional but recommended.

**Q: What happens after I submit?**
A: Your submission is locked and appears in manager's report immediately.

**Q: Can manager see my draft?**
A: No, only after you submit.

**Q: How many user stories should I add?**
A: Add all stories you worked on (no limit).
