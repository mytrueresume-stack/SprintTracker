# ?? Sprint Management - Quick User Guide

## For Managers

### Starting a Sprint
1. Go to your project (Dashboard ? Projects ? Select Project)
2. Click **"Sprints"** tab
3. Find the sprint with **"Planning"** status
4. Click **"Start"** button
5. ? Sprint is now active and team members can submit their work

### Completing a Sprint
1. Go to your project
2. Click **"Sprints"** tab
3. Find the **active sprint** (green "Active" badge)
4. Click **"Complete"** button
5. Confirm the action
6. ? Sprint is marked as completed with velocity calculated

### Creating a Sprint
1. Go to your project
2. Click **"Sprints"** tab
3. Click **"Create Sprint"** button
4. Fill in:
   - Sprint Name (e.g., "Sprint 1", "January 2026 Sprint")
   - Sprint Goal (optional but recommended)
   - Start Date
   - End Date
5. Click **"Create Sprint"**
6. ? Sprint is created in "Planning" status

### Managing Team
1. Go to your project
2. Click **"Manage Team"** button (top right)
3. Add or remove team members
4. Click **"Save"**

## For Developers

### Viewing Active Sprint
- Check your **Dashboard** - active sprints show at the top
- Or go to **Projects** ? Select project ? See active sprint banner

### Submitting Sprint Work
#### Option 1: From Dashboard
1. See active sprint card on dashboard
2. Click **"Submit Work"** button

#### Option 2: From Project Page
1. Go to your project
2. See active sprint banner at top
3. Click **"Submit Work"** button

#### Option 3: From Sprints Tab
1. Go to your project
2. Click **"Sprints"** tab
3. Find active sprint
4. Click **"Submit Work"** button

### Filling Submission Form
1. **Work Summary** (Required)
   - Story Points Planned
 - Story Points Completed
   - Hours Worked

2. **User Stories** (Optional but recommended)
   - Add stories you completed
   - Story ID (e.g., PROJ-123)
   - Title and points
   - Status

3. **Achievements** (Optional)
   - What did you accomplish?

4. **Learnings** (Optional)
   - What did you learn?

5. **Next Sprint Goals** (Optional)
   - What will you work on next?

6. Click **"Save Draft"** to save progress (can edit later)
7. Click **"Submit"** when ready (cannot edit after submission)

### Checking Your Tasks
1. Dashboard ? See "Recent Tasks" section
2. Or click **"My Tasks"** card
3. Update task status as you work

## Common Issues & Solutions

### ? "You don't have permission to start sprint"
**Solution**: Only managers and project owners can start sprints. Contact your manager.

### ? "No active sprint" message
**Solution**: Wait for your manager to start a sprint, or contact them if needed.

### ? Cannot edit sprint submission
**Solution**: Once submitted, submissions cannot be edited (by design). Save as draft if you need to make changes.

### ? "Submit Work" button not showing
**Solution**: 
- Check if sprint is actually active (not Planning or Completed)
- Refresh the page
- Contact your manager if issue persists

## Tips & Best Practices

### For Managers
- ? Set clear sprint goals before starting
- ? Add all team members to project before starting sprint
- ? Review team submissions regularly
- ? Complete sprint promptly at end date
- ? Review retrospective data to improve future sprints

### For Developers
- ? Submit your work before sprint ends
- ? Update task status regularly (not just at sprint end)
- ? Be honest about story points completed
- ? Document learnings and impediments
- ? Save draft if you're not done yet
- ? Review your submission before final submit

## Quick Reference - Button Colors

| Button | Color | Meaning |
|--------|-------|---------|
| Start Sprint | Blue outline | Action available |
| Complete Sprint | Green outline | Completion action |
| Submit Work | Blue filled | Primary developer action |
| Save Draft | Gray outline | Non-destructive save |
| Submit | Blue filled | Final submission |

---

## Status Badges

| Status | Color | Meaning |
|--------|-------|---------|
| Planning | Yellow | Sprint not started yet |
| Active | Green (with dot) | Sprint in progress |
| Completed | Gray | Sprint finished |
| Submitted | Green | Work submission completed |
| Draft | Yellow | Submission in progress |

## Need Help?

1. **Check the dashboard** - Most actions are available there
2. **Look for green badges** - They indicate active/available items
3. **Read error messages carefully** - They usually explain the issue
4. **Contact your manager** - For permission or project-related issues
5. **Check your role** - Some features are manager-only
